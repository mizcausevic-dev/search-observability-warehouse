Warehouse Layout

The warehouse separates stable business entities from volatile search facts, then recombines them in rollup views built for diagnostics.

sites

  • domain
  • market
  • site code

page_groups

  • template
  • owner lane
  • group key

search_console_daily

  • impressions
  • clicks
  • CTR
  • position

index_coverage_daily

  • indexed
  • excluded
  • canonical conflicts

crawl_daily

  • response ms
  • error rate
  • robots blocks

url_signals

  • crawl status
  • index status
  • response code

source_freshness

  • last loaded at
  • SLA hours
  • row count

anomaly_flags

  • severity
  • impact metric
  • action

Rollup views

`vw_site_overview`, `vw_page_group_rollup`, `vw_freshness_gaps`, and `vw_crawl_index_mismatches` provide the operator layer.

Diagnostic intent

The shape is optimized for explaining KPI movement, not just reporting it. Freshness drift, exclusions, and crawl failures can be traced into the same ownership lane.
